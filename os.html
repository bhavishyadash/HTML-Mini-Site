What an OS Does

  • Process management: schedules work and isolates programs.
  • Memory management: virtual addresses, paging, protection.
  • File systems: organizes data into directories and files.
  • Device drivers & I/O: unifies access to hardware.
  • Security: users, permissions, and sandboxing.

Userland & System Calls

Applications run in user space and ask the kernel for services using system calls (e.g., open, read, write). POSIX standardizes many of these, which is why code ports across Unix-like systems fairly well.
