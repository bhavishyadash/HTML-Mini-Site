SDLC Overview

Purpose

Purpose

SDLC is a structured approach for building and maintaining software. It reduces risk by defining stages and checkpoints for quality, budget, and schedule.

Key Outcomes by Stage

  • Requirements: scope, acceptance criteria.
  • Design: architecture, data models, interfaces.
  • Implementation: code, reviews, CI builds.
  • Testing: unit/integration tests, bug tracking.
  • Deployment: release plan, rollout/rollback.
  • Maintenance: monitoring, patches, improvements.

Common Methodologies

Common Methodologies

  • Waterfall: linear phases; predictable, weak on late changes.
  • Agile (Scrum/Kanban): iterative delivery and feedback.
  • Spiral: risk-driven iterations for high uncertainty.
  • V-Model: verification/validation mapped to each phase.

Choosing a Method

  • Stable requirements & compliance → Waterfall / V-Model.
  • Changing requirements & user feedback → Agile.
  • High technical risk or novel R&D → Spiral.

Process + Practice

Methodology frames when you do work; engineering practices raise how well you do it: version control, CI/CD, automated testing, code review, observability, and incident response.
